# SEO & E-E-A-T Standards

## E-E-A-T Framework

Google's quality framework evaluates content on four dimensions. **Trust** is the most important—it's built through the other three.

### Experience
Demonstrate first-hand, real-world experience with your topic.

**How to show Experience:**
- Use "I tested..." or "In our experience..." language
- Include original photos/screenshots from actual use
- Share specific results, numbers, timelines
- Describe challenges encountered and how you solved them
- Reference how long you've worked with/used the subject

**Example signals:**
- "After using this tool for 6 months, here's what I found..."
- "We implemented this strategy with 12 clients last quarter..."
- Original case study data from your own work

### Expertise
Show deep knowledge and competence in the subject area.

**How to show Expertise:**
- Display relevant credentials (certifications, education)
- Link to other authoritative content you've created
- Use accurate, current terminology
- Go deeper than surface-level advice
- Anticipate and address edge cases

**Example signals:**
- Detailed author bio with relevant background
- Citations of primary research
- Technical accuracy verified by subject matter experts
- Nuanced perspectives that show real understanding

### Authoritativeness
Establish your reputation as a go-to source.

**How to show Authoritativeness:**
- Get mentioned/cited by other reputable sources
- Build a content hub around your expertise areas
- Include quotes from recognized industry experts
- Reference industry awards, recognition, or publications
- Maintain consistent publishing in your niche

**Example signals:**
- External backlinks from authoritative domains
- Media mentions and press coverage
- Industry association memberships
- Speaking engagements and published works

### Trustworthiness
Build confidence that your content is honest and reliable.

**How to show Trustworthiness:**
- Clearly disclose affiliations, sponsorships, ads
- Cite sources for all factual claims
- Keep content current (show last-updated dates)
- Provide contact information
- Have clear privacy policy and terms
- Correct errors publicly and promptly

**Example signals:**
- "Last updated: [date]" on all content
- "Disclosure: We may earn commission..." where applicable
- Links to authoritative sources for statistics
- Easy-to-find contact and about pages

## On-Page SEO Checklist

### Title Tag
- [ ] 50-60 characters (will truncate at ~60)
- [ ] Primary keyword included (front-loaded when natural)
- [ ] Compelling and click-worthy
- [ ] Unique across the site
- [ ] Brand name at end (if space permits)

**Formula**: `[Primary Keyword]: [Benefit/Promise] | [Brand]`

### Meta Description
- [ ] 150-160 characters
- [ ] Includes primary keyword naturally
- [ ] Has clear value proposition
- [ ] Includes call-to-action
- [ ] Matches search intent

### URL Structure
- [ ] Short and descriptive (3-5 words ideal)
- [ ] Includes primary keyword
- [ ] Uses hyphens, not underscores
- [ ] Lowercase only
- [ ] No dates (unless news content)

**Good**: `/content-quality-checklist`
**Bad**: `/2024/01/15/the-ultimate-content-quality-checklist-for-bloggers`

### Heading Structure
- [ ] Single H1 (the title)
- [ ] H2s for main sections
- [ ] H3s for subsections under H2s
- [ ] Logical hierarchy (no skipping levels)
- [ ] Keywords in headings where natural
- [ ] Headings are descriptive (not "Introduction")

### Content Optimization
- [ ] Primary keyword in first 100 words
- [ ] Primary keyword density 1-2% (natural, not forced)
- [ ] Secondary/related keywords distributed naturally
- [ ] Semantic variations used throughout
- [ ] Questions from "People Also Ask" addressed
- [ ] Content length competitive with ranking pages

### Internal Linking
- [ ] 3-5 internal links minimum
- [ ] Anchor text descriptive (not "click here")
- [ ] Links to relevant, high-value pages
- [ ] Links placed naturally in context
- [ ] No orphan pages created

### External Linking
- [ ] Links to authoritative sources for claims
- [ ] 2-5 external links typical
- [ ] Links open in new tab (rel="noopener")
- [ ] No links to competitors (unless comparison content)
- [ ] Avoid linking to low-quality sites

### Images
- [ ] Descriptive file names (`content-checklist-example.webp`)
- [ ] Alt text describes image AND includes keyword where relevant
- [ ] Compressed (<200KB, prefer WebP)
- [ ] Responsive/properly sized
- [ ] Featured image optimized for social sharing

## Content Freshness

### When to Update Content
- Core information has changed
- Statistics are >2 years old
- Rankings have dropped significantly
- New developments in the topic
- User feedback indicates confusion
- Competitor content has surpassed yours

### Update Checklist
- [ ] Review and update statistics
- [ ] Add new sections for emerging subtopics
- [ ] Remove outdated information
- [ ] Refresh examples and screenshots
- [ ] Update internal links
- [ ] Revise meta description if needed
- [ ] Update "Last modified" date
- [ ] Don't change URL unless absolutely necessary

## Schema Markup

Implement appropriate schema for content type:

| Content Type | Schema Type |
|--------------|-------------|
| Blog post | Article, BlogPosting |
| How-to guide | HowTo |
| FAQ content | FAQPage |
| Review | Review |
| List/Roundup | ItemList |
| Recipe | Recipe |
| Event | Event |

### Required Article Schema Fields
- headline
- author (with @type: Person or Organization)
- datePublished
- dateModified
- image
- publisher

## Search Intent Alignment

Match content format to user intent:

| Intent | User Wants | Content Format |
|--------|------------|----------------|
| Informational | Learn something | Guides, explainers, definitions |
| Navigational | Find specific page/site | Landing pages, about pages |
| Transactional | Complete action | Product pages, signup pages |
| Commercial | Compare before buying | Reviews, comparisons, lists |

**Always check**: Search your target keyword and see what's ranking. Match that format and intent—then do it better.
