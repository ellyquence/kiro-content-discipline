# Content Quality Gates

## Overview

Quality gates are mandatory checkpoints that content must pass before advancing to the next stage. No content moves forward without meeting gate requirements.

## Gate 1: Brief Approved

**Purpose**: Ensure content is strategically aligned before writing begins.

### Requirements:
- [ ] Target audience persona identified
- [ ] Primary keyword/topic defined
- [ ] Search intent understood (informational/navigational/transactional)
- [ ] Content goal specified (traffic/leads/authority/engagement)
- [ ] Competitor content analyzed (top 3-5 ranking pieces)
- [ ] Unique angle or value-add defined
- [ ] Word count target set
- [ ] Structure framework chosen (AIDA/PAS/How-To/List)
- [ ] Voice/tone direction confirmed

### Approval Required From:
- Content strategist or editor

---

## Gate 2: Outline Ready

**Purpose**: Validate structure before full draft investment.

### Requirements:
- [ ] 25+ headline variants brainstormed
- [ ] Top 3 headlines scored/tested
- [ ] H2 structure covers topic comprehensively
- [ ] H3 subsections logically organized
- [ ] Introduction hook defined
- [ ] Key points/arguments listed under each section
- [ ] Sources identified for claims
- [ ] CTA determined and placed
- [ ] Internal linking opportunities noted
- [ ] Estimated reading time calculated

### Approval Required From:
- Editor or content lead

---

## Gate 3: First Draft Complete

**Purpose**: Ensure draft meets baseline quality before editorial review.

### Self-Review Checklist:
- [ ] All outline sections written
- [ ] Word count within 10% of target
- [ ] Introduction hooks reader in first 100 words
- [ ] Each section delivers on its heading promise
- [ ] Transitions connect sections smoothly
- [ ] Claims supported with evidence/sources
- [ ] Examples or case studies included
- [ ] Conclusion summarizes and prompts action
- [ ] CTA is clear and compelling

### Technical Checks:
- [ ] Spell check passed
- [ ] Grammar check passed
- [ ] Readability score â‰¤ Grade 8
- [ ] Passive voice < 10%
- [ ] No orphaned headings (heading with no content)
- [ ] Links tested and working

---

## Gate 4: Editorial Review

**Purpose**: Professional review for quality, accuracy, and brand alignment.

### Editor Checklist:

**Voice & Tone**:
- [ ] Matches brand voice guidelines
- [ ] Tone appropriate for content type
- [ ] Consistent throughout piece
- [ ] No jarring style shifts

**Clarity & Structure**:
- [ ] Headline accurately reflects content
- [ ] Subheadings scannable and descriptive
- [ ] Paragraphs focused (one idea each)
- [ ] Logical flow from intro to conclusion
- [ ] No redundant sections

**Accuracy & Credibility**:
- [ ] Facts verified against original sources
- [ ] Statistics current (within 2 years)
- [ ] Quotes accurately attributed
- [ ] No unsupported claims
- [ ] Expertise demonstrated (E-E-A-T)

**Engagement**:
- [ ] Hook captures attention
- [ ] Value delivered early
- [ ] Reader questions anticipated and answered
- [ ] Emotional resonance present
- [ ] CTA compelling and clear

### Revision Feedback Format:
```
[SECTION]: [Issue Type]
Current: "[exact text]"
Issue: [what's wrong]
Suggestion: "[proposed fix]"
Priority: High/Medium/Low
```

---

## Gate 5: SEO & Technical Review

**Purpose**: Ensure content is discoverable and technically sound.

### On-Page SEO:
- [ ] Primary keyword in title (front-loaded if possible)
- [ ] Primary keyword in first 100 words
- [ ] Primary keyword in at least one H2
- [ ] Secondary keywords naturally distributed
- [ ] Meta title optimized (50-60 characters)
- [ ] Meta description compelling (150-160 characters)
- [ ] URL slug clean and keyword-rich
- [ ] Image alt text descriptive and keyword-aware

### E-E-A-T Signals:
- [ ] Author byline present
- [ ] Author bio with credentials
- [ ] Publication date visible
- [ ] Last updated date (if applicable)
- [ ] Sources linked to authoritative sites
- [ ] Expert quotes included (if applicable)

### Technical:
- [ ] All links working (internal and external)
- [ ] Images optimized (<200KB, WebP preferred)
- [ ] Featured image sized correctly
- [ ] Schema markup appropriate for content type
- [ ] Mobile preview checked
- [ ] Page load acceptable

---

## Gate 6: Final Approval

**Purpose**: Last check before scheduling/publication.

### Final Checklist:
- [ ] All previous gate requirements met
- [ ] No placeholder text remaining
- [ ] Preview matches expected layout
- [ ] Social sharing snippets ready
- [ ] Publication time optimized
- [ ] Promotion plan in place
- [ ] Team notified of publication

## Gate Failure Protocol

If content fails a gate:

1. **Document** the specific failures
2. **Return** to appropriate stage with feedback
3. **Revise** addressing all noted issues
4. **Resubmit** for gate review
5. **Track** failure patterns for process improvement

## Quality Metrics Dashboard

Track these metrics monthly:
- Gate pass rate by stage
- Average revision cycles
- Time per stage
- Most common failure reasons
- Post-publish performance vs. gate-passed content
